# AI 重绘 · 订阅消息推送 · 接入指南

> 本次改造做了一件事：AI 重绘任务被切到后台之后，做完了**通过订阅消息把用户拉回来看图**。
> 用户体感：点完按钮 → 弹"AI 已开始绘制，要继续等还是稍后通知？" → 选稍后 → 关掉小程序去玩别的 → 几十秒后微信收到通知 → 点开自动展示已生成的海报。

整个推送链路要跑起来，**你只要在微信小程序后台做两件事 + 在云托管加一个环境变量 + 在前端代码里填一行模板 ID**。详细步骤如下。

---

## 一、申请订阅消息模板（5 分钟）

### Step 1：进后台

1. 浏览器打开 [https://mp.weixin.qq.com](https://mp.weixin.qq.com)
2. 用绑定的微信扫码登录
3. 左侧菜单：**功能 → 订阅消息**

### Step 2：选模板

1. 点 **「公共模板库」** 标签
2. 搜索关键词（任选其一）：`完成通知` / `处理完成` / `生成完成` / `任务通知`
3. 推荐挑这种结构的模板（字段顺序最重要）：

   | 字段顺序 | 字段类型 | 我们要塞的内容          | 示例                  |
   |---------|---------|----------------------|---------------------|
   | thing1  | 短文本   | 宠物名 / 标题          | "猫主子"             |
   | thing2  | 短文本   | 完成结果              | "水彩童画 海报已生成" |
   | time3   | 时间     | 完成时间              | "2026年4月30日 18:30" |
   | thing4  | 短文本   | 备注                  | "点击查看，30 分钟内有效" |

   找一个**前 4 个字段就是 thing-thing-time-thing** 的模板（字段类型对就行，字段编号不一定是 1234）。

   > **如果你找的模板字段顺序不一样**，没关系，记下你模板里的真实字段名（例如 `phrase5` 而不是 `thing2`），告诉我，我把后端 `wx_subscribe.py` 里的 `build_redraw_done_data()` 改一下字段名就行。

4. 点 **「选用」** → 模板被加到 **「我的模板」** 里
5. 在「我的模板」标签页，复制刚选用的模板的 **「模板 ID」**（一长串大小写字母数字，形如 `8Wxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

### Step 3：保存模板 ID 到两个地方

#### 地方 A：云托管环境变量（后端用）

1. [https://cloud.weixin.qq.com](https://cloud.weixin.qq.com) → 你的环境 → 云托管 → 服务列表
2. 找到 photo 服务 → 「服务设置」→「环境变量」
3. **新增**一条：

   | 变量名                    | 变量值                      |
   |--------------------------|---------------------------|
   | `WX_REDRAW_TEMPLATE_ID`  | 粘贴你刚复制的模板 ID         |

4. 保存 → 点 **「重新部署」**（环境变量改了必须重启实例才生效）

#### 地方 B：前端代码（订阅授权弹窗用）

打开 `miniprogram/pages/photo/photo.js`，找到顶部第 14 行附近：

```javascript
const REDRAW_NOTIFY_TMPL_ID = "";  // ← TODO: 填模板 ID 后启用推送
```

把双引号里填上你的模板 ID：

```javascript
const REDRAW_NOTIFY_TMPL_ID = "8Wxxxxxxxxxxxxxxxxxxxxxxxxxxx";
```

保存 → 微信开发者工具 → 上传体验版 → 提审正式版 → 等通过。

## 三、用户体验是这样的

### 没配模板 ID（现在的状态）
- 用户点 [AI 重绘] → 弹"AI 已开始绘制，预计 30 秒"
- 选 [继续等待] → 走原来的轮询出图流程
- 选 [稍后通知我] → 任务在后台跑 + Toast "AI 在后台绘制，下次回来自动展示"
- 用户下次进入 photo 页 → 自动拉一次 → 出图

### 配好模板 ID 后
- 同上，但是用户点 [AI 重绘] 时**先弹一个微信原生订阅授权框**："允许这个小程序给你发一次完成通知吗？"
- 用户点同意 → 上述流程不变，但**任务做完会主动推一条订阅消息到微信**
- 用户在微信里看到通知 → 点进去 → 直接落到 photo 页 + 自动展示已生成的图（带的 `?task_id=xxx` 参数让 onShow 知道优先拉哪个任务）

## 四、配额 & 频次说明

| 项目                              | 限制                                                       |
|-----------------------------------|----------------------------------------------------------|
| 用户每授权一次，能推几次？           | **1 次**。微信订阅消息是"一次性"的，每次重绘都要重新授权        |
| 用户能勾"始终允许"避免每次弹吗？     | 可以。只要他在弹窗下面勾了「总是保持以上选择」                  |
| 推送失败会怎样？                   | 默默吞掉 + 后端 warning 日志。任务结果照常存，用户进小程序还是能看到 |
| 用户拒绝订阅了，能用 AI 重绘吗？     | 能。回退到"任务在后台跑，下次进小程序自动展示"流程               |
| 模板 ID 没配，会报错吗？            | 不会。前端直接跳过授权步骤，回退到全程等待 / 稍后展示流程         |

## 六、常见问题

**Q: 模板审核要多久？**
A: 公共模板库里选用的，**实时生效**，不审核。只有你提交"个人定制模板"才走审核（不推荐，门槛高）。

**Q: 我能不能不用模板，自己写文案？**
A: 不能。微信对小程序订阅消息强约束模板字段，每个 `thing` 字段最多 20 字符、`time` 必须标准时间格式。这是为了防滥用。

**Q: 用户没收到推送，怎么排查？**
A: 云托管「运行日志」搜 `[subscribe]`：
- `sent ok` → 微信已发，可能用户没开微信通知 / 进了"服务通知"折叠列表
- `user did not subscribe (43101)` → 用户那次没同意订阅
- `WX_REDRAW_TEMPLATE_ID 未配置` → 环境变量没生效，重新部署
- `errcode=47003` → 字段名不匹配，对照模板定义改 `wx_subscribe.py:build_redraw_done_data()`

**Q: 推送一次的费用？**
A: 订阅消息**完全免费**，无论推多少。微信不收钱。
